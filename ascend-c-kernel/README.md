# Ascend C TQ3 Dequant Kernel (Ready for Compilation)

## Status

The custom Ascend C kernel for TQ3 dequantization is **implemented and ready for msopgen compilation**. We have:

- ✅ Generated the msopgen project skeleton
- ✅ Written the operator descriptor (`op_desc.json`) with multidim shape support
- ✅ Written the host-side operator code (`tq3_dequant_host.cpp`) with correct InferDataType and InferShape
- ✅ Implemented the device kernel body (`tq3_dequant_kernel.cpp`) — uses Ascend C DataCopy, LocalTensor, TPipe/TQue pipeline
- ✅ Fixed output type to float16 and output shape transform (output_dim = input_dim * 2)

## Architecture

```
op_desc.json → msopgen → cmake → opbuild → Tq3Dequant.om
```

## File Layout

```
ascend-c-kernel/
├── op_desc.json                     # Operator description (input: uint8 TQ3 tensor, output: float16 weights)
├── src/
│   ├── tq3_dequant_host.cpp         # Host-side: operator registration, tiling, InferShape/DataType
│   └── tq3_dequant_kernel.cpp       # Device kernel: Ascend C kernel with DataCopy pipeline
```

## Compilation

Use the msopgen cmake flow:

```bash
msopgen gen -c op_desc.json -lang cpp
cd build
cmake .. -DASCEND_TOOLKIT_HOME=${ASCEND_TOOLKIT_HOME}
cmake --build .
```

The device kernel `.o` files are generated by `opbuild` as part of the cmake build.

## Known Limitations

- The current kernel uses scalar float operations on device; vectorized SIMD ops (vadd, vmul, etc.) would improve throughput
- The kernel processes one block at a time; block-level parallelism could be improved
- The tiling function currently assumes 1D flattened input — needs update for 2D tensor shapes
